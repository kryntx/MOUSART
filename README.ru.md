**MOUSART** — полнофункциональный отладчик последовательных портов, созданный на Python/PyQt6, предназначенный для встраиваемой разработки, отладки аппаратного обеспечения и последовательной связи. Версия 3.0 — это полная переработка на Python, которая добавляет автоответ, поддержку протокола Modbus, быстрые команды, запись и экспорт данных, управление выводами, поддержку нескольких кодировок и десятки профессиональных функций.

## Обзор функций

### 1. Управление подключениями
- Автообновление списка доступных портов (каждые 2с)
- Виртуальный последовательный порт, USB-адаптер, Bluetooth
- Скорость: 300–921600 предустановки + пользовательская (1-9999999)
- Биты данных: 5, 6, 7, 8 / Стоп-биты: 1, 1.5, 2
- Чётность: None, Even, Odd, Mark, Space
- Управление потоком: Нет / Аппаратное (RTS/CTS) / Программное (XON/XOFF)
- Ручное управление DTR/RTS / Мониторинг CTS/DSR/DCD/RI

### 2. Приём данных
- Текстовый режим (UTF-8 / GBK / GB18030 / Latin-1 / ASCII)
- Шестнадцатеричный (HEX) режим
- Временная метка (точность до миллисекунды)
- Индикатор направления (`<<` RX / `>>` TX / `!!` ERR)
- Пауза отображения (приём продолжается в фоне)
- Фильтр по ключевому слову (текст / regex)
- Лимит журнала: 2000 записей

### 3. Отправка данных
- Режим отправки Текст / HEX
- Управление переводом строки (CR / LF независимо)
- Периодическая отправка (1мс–3600000мс, с ограничением количества)
- Отправка файла
- Конструктор кадра Modbus RTU (автоматический CRC)
- Панель быстрых команд (добавление/редактирование/удаление,持久ный)
- Последовательность/очередь отправки (поддержка цикла)

### 4. Автоответ
- Автоматический ответ при получении определённого ключевого слова
- Настраиваемая задержка ответа (мс)
- Включение/отключение одним щелчком

### 5. Кодирование и конвертация
- Конвертация Текст ↔ Hex
- Множественные кодировки: UTF-8, GBK, GB18030, Latin-1, ASCII
- Контрольные суммы: Sum8 / XOR8 / CRC16-Modbus / CRC32
- Конструктор и парсер кадра Modbus RTU
- Конвертация двоичное / десятичное / шестнадцатеричное

### 6. Запись и экспорт
- Сохранение журнала в TXT / CSV
- Автоматическая запись (файл 10МБ)
- Счётчик байтов RX/TX в реальном времени

### 7. Виртуальный последовательный порт (только Linux)
- Создание виртуального порта через `socat`
- Подключение внешних программ через `/tmp/mousart_vport`

### 8. Интерфейс и конфигурация
- Тёмная / Светлая тема
- Масштабирование шрифта 0.8x–1.5x
- Управление профилями (несколько конфигураций)

## Быстрый старт

### Готовые бинарные файлы

Скачать с [GitHub Releases](https://github.com/kryntx/MOUSART/releases):

| Платформа | Файл | Размер |
|-----------|------|--------|
| **Linux x86_64 (deb)** | `mouserial_3.0.0-1_amd64.deb` | ~82KB |
| **Windows x86_64** | `MOUSART-v3.0.0-windows-x86_64.exe` | ~23МБ |

#### Установка Debian/Ubuntu (Рекомендуется)
```bash
# Скачать и установить deb-пакет (зависимости обрабатываются автоматически)
wget https://github.com/kryntx/MOUSART/releases/download/v3.0.0/mouserial_3.0.0-1_amd64.deb
sudo apt install ./mouserial_3.0.0-1_amd64.deb
```

### Сборка из исходного кода

```bash
# Ubuntu/Debian зависимости
sudo apt install python3-pyqt6 python3-serial socat

git clone https://github.com/kryntx/MOUSART.git
cd MOUSART

# Установить зависимости Python
pip3 install PyQt6 pyserial

# Запуск
python3 -m mousart
```

#### Сборка EXE для Windows

```bash
pip3 install pyinstaller
pyinstaller pyinstaller.spec --noconfirm
# Выход: dist/MOUSART.exe
```

#### Сборка пакета Debian

```bash
sudo apt install dh-python python3-setuptools debhelper
dpkg-buildpackage -us -uc -b
```

## Использование

### Режим виртуального последовательного порта
1. Выбрать режим **"Virtual"** в левой панели
2. Нажать **"Start"**
3. Отображается путь к порту `/tmp/mousart_vport`
4. Подключить любую последовательную утилиту к этому пути

### Режим аппаратной отладки
1. Выбрать режим **"Debug"**
2. Выбрать порт и настроить параметры
3. Нажать **"Open"** для подключения
4. Ввести данные в область отправки и нажать **"Отправить"** или `Ctrl+Enter`

### Быстрые команды
1. Нажать **"+"** на панели быстрых команд
2. Ввести имя и данные
3. Левый клик — отправить, правый клик — редактировать

### Автоответ
1. Раздел **"Автоответ"** в левой панели
2. Включить переключатель, задать ключевое слово и данные ответа

### Конструктор Modbus
1. Нажать **"Modbus"** на панели инструментов
2. Задать адрес ведомого, код функции, начальный адрес, количество
3. Нажать **"Создать"** для автоматической генерации кадра с CRC

## FAQ

**Q: Не удаётся открыть порт?** Проблема с правами доступа. Используйте `sudo python3 -m mousart` или добавьте пользователя в группу `dialout`.

**Q: Виртуальный порт не работает?** Установите `socat`: `sudo apt install socat`
