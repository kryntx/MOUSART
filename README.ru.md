**MOUSART** - это полнофункциональный инструмент отладки последовательных портов, предназначенный для встроенной разработки, отладки оборудования и сценариев последовательной связи. Поддерживает два независимых режима: отладку последовательных портов и виртуальный последовательный порт, с десятками профессиональных функций, таких как автоматический ответ, поддержка протокола Modbus, быстрые команды, запись и экспорт данных, управление выводами и поддержка нескольких кодировок.

## Функции

### Режим отладки последовательных портов
- Автоматическое обновление списка доступных последовательных портов (опрос каждые 2 секунды)
- Поддержка USB-serial адаптеров, Bluetooth serial и т.д.
- Скорость передачи: 300 ~ 921600 предустановленных значений + пользовательские значения (1-9999999)
- Биты данных: 5, 6, 7, 8 / Стоповые биты: 1, 1.5, 2
- Четность: None, Even, Odd, Mark, Space
- Управление потоком: Нет / Аппаратное (RTS/CTS) / Программное (XON/XOFF)
- Ручное управление DTR/RTS
- Мониторинг уровней выводов CTS/DSR/DCD/RI в реальном времени
- Автоматический ответ (совпадение ключевых слов + настройка задержки)

### Режим виртуального последовательного порта (только Linux)
- Создание пары виртуальных последовательных портов в один клик на основе `socat`
- Внешние программы подключаются через `/tmp/mousart_vport`
- Независимые области отправки/приема и статистика

### Отправка и прием данных
- Переключение режима отправки Текст / HEX
- Поддержка нескольких кодировок: UTF-8, GBK, GB18030, Latin-1, ASCII
- Управление переводом строки при отправке (независимые переключатели CR / LF)
- Отправка по таймеру (1мс ~ 3600000мс)
- Отправка файла
- Конструктор кадров Modbus RTU (автоматический CRC)
- Панель быстрых команд (левый клик отправить, правый клик редактировать)
- Горячая клавиша Ctrl+Enter для отправки

### Анализ данных
- Расчет контрольных сумм: Sum8 / XOR8 / CRC16-Modbus / CRC32
- Преобразование двоичное / десятичное / шестнадцатеричное
- Преобразование Текст <-> Hex в реальном времени
- Фильтрация по ключевому слову / регулярному выражению

### Запись данных
- Сохранение журналов в TXT / CSV в один клик
- Автоматическая запись (автоматическое разделение на 10 МБ/файл)
- Счетчик байтов RX/TX в реальном времени и отображение скорости

### Интерфейс
- Двойная тема: темная/светлая
- Масштабирование шрифта 0.8x-1.5x
- Управление профилями

## Руководство по установке

### Linux (Debian/Ubuntu)

#### Способ 1: Установка deb-пакета (рекомендуется)

```bash
# 1. Скачать deb-пакет
wget https://github.com/kryntx/MOUSART/releases/download/v3.0.0/mouserial_3.0.0-1_amd64.deb

# 2. Установить (зависимости устанавливаются автоматически)
sudo apt install ./mouserial_3.0.0-1_amd64.deb

# 3. Запустить
mousart
```

#### Способ 2: Запуск из исходного кода

```bash
# 1. Установить системные зависимости
sudo apt update
sudo apt install python3-pyqt5 python3-serial socat git

# 2. Клонировать проект
git clone https://github.com/kryntx/MOUSART.git
cd MOUSART

# 3. Запустить
python3 -m mousart
```

#### Способ 3: Установить зависимости через pip и запустить

```bash
# 1. Установить системные зависимости
sudo apt install socat

# 2. Установить зависимости Python
pip3 install PyQt5 pyserial

# 3. Клонировать и запустить
git clone https://github.com/kryntx/MOUSART.git
cd MOUSART
python3 -m mousart
```

## Руководство по использованию

### Режим отладки последовательных портов
1. В левой панели выберите режим **"Отладка последовательных портов"**
2. Выберите последовательный порт из выпадающего списка
3. Настройте параметры последовательного порта (скорость передачи, биты данных, стоповые биты, четность, управление потоком)
4. Нажмите **"Открыть"** для установки соединения
5. Введите данные в области отправки справа и нажмите **"Отправить"** или `Ctrl+Enter`
6. Область приема отображает полученные данные в реальном времени

### Режим виртуального последовательного порта (только Linux)
1. В левой панели выберите режим **"Симуляция последовательного порта"**
2. Нажмите **"Запустить"** для создания пары виртуальных последовательных портов
3. Интерфейс отображает путь внешнего порта `/tmp/mousart_vport`
4. Подключитесь к этому пути с помощью любого инструмента для работы с последовательными портами
5. Введите данные в области отправки и отправьте

### Быстрые команды
1. Нажмите **"+"** на панели быстрых команд внизу, чтобы добавить новую команду
2. Введите имя и данные (поддерживается текст и HEX)
3. Левый клик по кнопке команды для немедленной отправки
4. Правый клик по кнопке команды для редактирования или удаления

### Автоматический ответ (Режим отладки последовательных портов)
1. Найдите область **"Автоматический ответ"** в левой панели
2. Активируйте переключатель и настройте ключевое слово для совпадения и данные ответа
3. При желании настройте задержку ответа (миллисекунды)

### Конструктор кадров Modbus
1. Нажмите кнопку **"Modbus"** на панели инструментов отправки
2. Настройте адрес ведомого, код функции, начальный адрес и количество
3. Нажмите **"Создать"** для автоматической генерации кадра Modbus RTU с CRC

### Запись и экспорт журналов
1. Нажмите кнопку **"Запись"** на панели инструментов области приема для начала автоматической записи
2. Все отправленные и полученные данные автоматически сохраняются в `~/mousart_logs/`
3. Нажмите **"Сохранить"** для ручного экспорта в TXT или CSV

### Управление выводами (Режим отладки последовательных портов)
1. После открытия последовательного порта в левой панели отображается панель управления выводами
2. Нажмите кнопки DTR/RTS для переключения уровня выводов
3. Индикаторы состояния CTS/DSR/DCD/RI отображаются в реальном времени

## Часто задаваемые вопросы

**Q: Не удается открыть последовательный порт?**
A: Обычно это проблема прав доступа. Добавьте пользователя в группу `dialout`:
```bash
sudo usermod -aG dialout $USER
# Выйдите и войдите снова, чтобы изменения вступили в силу
```

**Q: Виртуальный последовательный порт не работает?**
A: Убедитесь, что установлен `socat`:
```bash
sudo apt install socat
```

**Q: Китайские символы отображаются некорректно?**
A: Выберите правильную кодировку приема в левой панели (например, GBK).

**Q: Как отправить команду Modbus?**
A: Нажмите кнопку **"Modbus"**, заполните параметры и нажмите **"Создать"**. Данные кадра будут автоматически заполнены в области отправки.

**Q: Где сохраняются файлы автоматической записи?**
A: По умолчанию в `~/mousart_logs/`, формат имени файла: `mousart_rec_YYYYMMDD_HHmmss.log`.
